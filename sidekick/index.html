01  /  aim

Set a goal. Plug in RL-inspired verifiers.

Tell the agent what you're shipping. Plug in verifiers that score how far the working tree is from done.

  • Bring your own. Any LLM skill, script, or binary that returns a distance.
  • Pinned & trusted. Remote verifiers are sha256-pinned, TOFU.
sidekick.yaml
goal_source: prompt verifiers: - name: Architect type: agent direction: N llm: { agent: claude, model: haiku } - name: Test type: binary direction: E binary: { command: ["go", "test"] } - name: Security direction: S - name: Deployment direction: W

02  /  orient

A live compass for your agent.

Each verifier reports a distance from 0 (done) to 1. Sidekick plots it; the agent reads the same snapshot and course-corrects mid-session.

  • TUI & menu bar. Compass, sparkline, one-key toggles.
  • 5-point anchors. Scores snap to 0 · .25 · .5 · .75 · 1 so they're comparable.
  • Re-runs on edit. File-write hooks debounce-trigger only the verifiers you've enabled.
03  /  wire

Drop-in simple.

A reward signal your agent can actually read — minus the training loop. Just hooks and MCP.

  • Claude Code & Codex. Supported out of the box via agent hooks, MCP, and a single skill.
  • Per-worktree sockets. Parallelize per worktree, or run repos side-by-side.
